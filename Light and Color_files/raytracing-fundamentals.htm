Raytracing Fundamentals

Resources RenderMan University Fundamentals Shading
Tags : introduction, raytracing, ver4

Ray tracing is a rendering technique that fires rays of light from the camera backwards through the viewing plane and into the scene. Once in the scene, the ray tests for intersections against a database of objects.
When the ray hits an object, the material properties of that object are evaluated. If the material has specular reflectivity, a continuation ray is traced in the reflection direction to test for reflected objects and return their illuminated color to the first surface. If the reflected ray fails to hit anything the environment or the background color is returned. Each time a ray hits an object it evaluates the local shading model, which usually requires illumination calculations. Often this can be done without ray tracing. However if raytraced shadows are required, a ray is fired in the direction of the light source. This ray tests for object intersections to determine if the surface is in shadow, partially illuminated, or fully illuminated.

In addition to specular reflections, there are also diffuse reflections. Diffuse reflections are found by sending out a hemisphere of rays above a surface point. Each diffuse ray returns either the color of the surface it hits or the color of the environment. All the rays are averaged to find the diffuse reflection. This hemispherical sampling technique is used to compute occlusion, colorbleeding, and image-based lighting (see IBL for more information).

What is a Diffuse Ray?

Technically, a diffuse ray is a ray that is cast in a hemisphere above a point where the angle of the hemisphere exceeds the angle specified by the RenderMan option specularthreshold. Practically, a diffuse ray is a ray used to sample effects that are view independent such as occlusion, colorbleeding, or image-based lighting. Typically they are spawned in a hemisphere above a shading point, (P), with the result being the average of the diffuse ray results (samples).

What is a Specular Ray?

Specular rays are typically cast for reflections and refractions. Any hemispherical rays that are fired for blurred reflection or refraction are categorized as specular if the cone angle is below the specular threshold option (described above). The limit on the number of rays that can be inter-reflected is set by the maxspeculardepth attribute below.

Ray Tracing Options and Attributes

RenderMan has a number of options and attributes that help tune the efficiency of ray tracing. The main attributes and options are described below with an extract from a RIB file that shows how the attribute/option manifests itself in RIB.

Trace Bias

This value offsets the origin of each ray by the specified amount of bias. The bias helps to overcome numerical precision issues, resulting in false ray-primitive intersections on the surface casting the rays. False intersections often appear as black patches along grid borders, but can be eliminated by increasing the trace bias.

	Attribute "trace" "float bias" [0.001]

Trace Depth

This option, set through the Render Settings, puts a maximum limit on ray-traced recursion (reflections of reflection of reflections...).

	Option "trace" "int maxdepth" [10]

Trace Displacements

This attribute controls whether displacements are considered during ray-primitives intersection tests. You can override this attribute on a per-object basis by adding the Trace displacements attribute. Note that tracing displacements are considerably slower. See Displacement II for more on tracing displacement.

	Attribute "trace" "int displacements" [1]

Max Specular Depth

This attribute sets the maximum number of bounces a specular ray can go through. Like all attributes this can be set on a per-object basis.

	Attribute "trace" "int maxspeculardepth" [2]

Max Diffuse Depth

This is the maximum number of bounces that a diffuse ray can perform. A diffuse ray is used for indirect illumination such as colorbleeding (where 1 ray is normally enough), as you just want to return the color of the object the ray hit on its first bounce.

	Attribute "trace" "int maxdiffusedepth" [1]

Ray Depth

Most of the attributes mentioned above regulate, the maximum traversal depth of rays. It is important to set this as small as possible to get the effect you are looking for. If too large, it is easy to spawn huge numbers of rays that contribute little to the overall image.
